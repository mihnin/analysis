ШАБЛОН 2: ИСТОРИЧЕСКИЕ ДАННЫЕ (полный) - РЕКОМЕНДУЕТСЯ!
================================================================================

Файл: historical_data_correct_template.xlsx
Колонок: 9
Статус: OK

Структура:
  1. Дата
  2. Филиал
  3. Материал
  4. Начальный остаток
  5. Поступление            <- ДОПОЛНИТЕЛЬНО!
  6. Потребление
  7. Конечный остаток
  8. Цена за единицу        <- ДОПОЛНИТЕЛЬНО!
  9. Конечная стоимость

Балансовое уравнение:
  Конечный остаток = Начальный остаток + Поступление - Потребление

Расчет стоимости:
  Конечная стоимость = Конечный остаток * Цена за единицу

Когда использовать:
  - Есть данные о поступлениях
  - Есть данные о цене за единицу
  - Нужна максимальная полнота и прозрачность

Результат валидации: КОРРЕКТЕН

ПРИМЕЧАНИЕ: Дополнительные колонки "Поступление" и "Цена за единицу"
            корректно обрабатываются приложением (не вызывают ошибок).

================================================================================
РЕКОМЕНДАЦИИ
================================================================================

ДЛЯ ТЕСТИРОВАНИЯ:
  1. Запустите: dist\Nornickel_Inventory_Analysis.exe
  2. Загрузите: datasets\historical_data_correct_template.xlsx
  3. Выберите: "Автоматический прогноз" (12 месяцев, AUTO)
  4. Нажмите: "Выполнить анализ"
  5. Сохраните результаты

ДЛЯ РАБОТЫ С РЕАЛЬНЫМИ ДАННЫМИ:
  - Используйте полный шаблон (historical_data_correct_template.xlsx)
  - Замените данные своими
  - Сохраните балансовое уравнение
  - Загрузите в приложение

ДЛЯ ПРОГНОЗА:
  - Рекомендуется: Режим "Автоматический прогноз" (не нужен файл!)
  - Альтернатива: Загрузите forecast_data_template.xlsx
